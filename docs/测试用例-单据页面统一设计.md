# 单据页面统一设计 — 手动测试用例

> 覆盖 6 个单据页面的功能验证，含具体操作步骤、预期结果、验证方法。

## 测试环境

- 后端: `cd backend && .venv/bin/uvicorn app.main:app --reload`
- 前端: `cd frontend && npm run dev`
- 数据库需有种子数据（产品: 蒙牛鲜奶/伊利酸奶，客户: 张老板超市/散客小王，供应商: 蒙牛代理）

## 验证方法说明

每条用例标注一种验证方法：

| 标识 | 方法 | 说明 |
|------|------|------|
| 👁️ 目视 | 肉眼确认 UI 展示 | 看页面元素是否与预期一致 |
| 🔢 数值 | 对比前后数值变化 | 操作前后记录数字，计算差值 |
| 🌐 网络 | DevTools Network 面板 | 看请求 status code 和 response |
| 🗄️ 数据库 | 直接查 DB | `sqlite3 backend/milk.db "SELECT ..."` |

## 快速冒烟测试（5 分钟）

按顺序快速过一遍核心路径，适合每次改动后跑：

1. **进货** → 新建 → 确认入库 1 条
2. **销售** → 新建 → 已收款 1 条
3. **退货** → 新建 → 关联来源 → 详情 → 撤销
4. **损耗** → 新建 → 详情 → 撤销
5. **送货** → 新建 → 详情 → 结算
6. **订奶** → 新建 → 点详情跳转

全部通过即核心功能无回归。
