# 送货单流程修正

## 背景

从送货单 #1（人民医院）发现三个问题：
1. 送货单创建时写入 `transaction (category=sale)`，和零售页面的 sale 混淆
2. 创建送货单不做库存校验，导致库存可为负
3. 换货 exchange 产生多余的 transaction，且金额变化后旧 transaction 对不上

## 设计

### 一、transaction 类别分离

送货单产生的交易 category 从 `"sale"` 改为 `"delivery"`：

| 类别 | 来源 | 含义 |
|------|------|------|
| `sale` | 零售页面 | 零售收入 |
| `delivery` | 送货单创建 | 应收发生 |
| `payment` | 结算还款 | 还款到账 |

所有流水集中在 transaction 表，AR 一表出结果。

### 二、exchange 更新 delivery transaction

换货时不再产生新 transaction，而是直接更新已有的 delivery transaction 金额：

- return_items：写 stock_movement（direction=in, reason=return）
- new_items：写 stock_movement（direction=out, reason=sale）
- 重新计算送货单金额后，更新 `delivery` transaction 的 `amount` 字段

### 三、应收公式修正

```
应收 = delivery - payment
```

```sql
SUM(CASE
  WHEN category = 'delivery' THEN amount
  WHEN category = 'payment' THEN -amount
  ELSE 0
END)
```

改动位置：`TransactionRepository.get_ar_by_customer` 和 `get_receivables`

### 四、送货库存校验

`create_delivery` 写入前检查每个品项的库存是否 ≥ 出货量，不足则 `raise ValueError`。

### 五、批量结算

新增 `POST /api/settlements/batch`：

```json
{
  "customer_id": 1,
  "items": [
    {"delivery_id": 1, "amount": 300},
    {"delivery_id": 3, "amount": 500}
  ]
}
```

前端：选客户 → 列出该客户所有未结清送货单 → 对每张单填写本次还款金额 → 提交。

### 六、已有数据修正

手动将数据库中送货单 #1 的 transaction category 从 `"sale"` 改为 `"delivery"`。
