# 全局重构设计文档

**日期**: 2026-06-10 **状态**: 设计完成，待实施

---

## 一、Context

当前系统问题：
- 各订单表自增 ID 独立，`source_id` 跨表不唯一，引用完整性靠代码兜底
- 品项信息混在 `StockMovement` 流水表里，查订单详情需要过滤 reason/cancel/exchange 等条件
- 流水表承载过多职责（品项详情 + 库存变动 + 溯源），字段冗余
- 缺少导入导出完整性

改造目标：Header-Detail 分离 + 全局唯一 ID + 精简流水表 + 导入导出全覆盖。

### 2.4 精简流水表

```python
class StockMovement(Base):
    __tablename__ = "stock_movements"
    id = Column(Integer, primary_key=True, autoincrement=True)
    product_id = Column(Integer, ForeignKey("products.id"), nullable=False)
    direction = Column(Enum(Direction), nullable=False)  # in / out
    quantity = Column(Integer, nullable=False)
    source_type = Column(Enum(DocumentType), nullable=True)  # 保留作查询缓存
    source_id = Column(Integer, ForeignKey("documents.id"), nullable=True)
    store_id = Column(Integer, ForeignKey("stores.id"), nullable=True)
    created_at = Column(DateTime, default=datetime.now)

class Transaction(Base):
    __tablename__ = "transactions"
    id = Column(Integer, primary_key=True, autoincrement=True)
    category = Column(Enum(TransactionCategory), nullable=False)
    amount = Column(Float, nullable=False)
    customer_id = Column(Integer, ForeignKey("customers.id"), nullable=True)  # AR 查询缓存
    source_type = Column(Enum(DocumentType), nullable=True)  # 查询缓存
    source_id = Column(Integer, ForeignKey("documents.id"), nullable=True)
    store_id = Column(Integer, ForeignKey("stores.id"), nullable=True)
    created_at = Column(DateTime, default=datetime.now)
```

> **`source_type` 保留决策**：去掉它后每次查 source 需 JOIN documents，高频路径（ledger 筛选、AR 查询）会多一层 JOIN。作为去正则化缓存保留。

## 四、order_number 生成

`next_order_number(db, Document, prefix)` → 查 documents 表过滤 `doc_type == prefix_type AND order_number LIKE '{prefix}{date}%'`，双重约束避免跨类型碰撞。逻辑不变。

### Settlement 路由

`settlements.py` 原先独立路由 `/api/deliveries/{id}/settle`，改为合并到 `distribution.py` API 里：
- `POST /api/distribution-orders/{id}/settle`
- `POST /api/distribution-orders/batch-settle`

删除 `settlements.py`。

前缀映射：
```
purchase → PO    retail → RO    distribution → DO    return → RT
wastage → WO    subscription → SO    store_sales → SS
```

### 导出列表（数据源切换）

| 模块 | 数据源 | CSV 列 |
|------|--------|--------|
| 进货 | purchase_items + documents + suppliers + products | 单号,供应商,日期,品名,数量,进价,金额,状态 |
| 零售 | retail_items + documents + customers + products | 单号,客户,日期,品名,数量,售价,折扣,实收,付款状态 |
| 铺货 | distribution_order_items + documents + customers + stores + products | 单号,客户,店铺,日期,品名,数量,售价,折扣,实收,付款状态 |
| 退货 | return_items + documents + customers + products | 单号,客户,日期,品名,数量,退款金额,关联铺货单 |
| 损耗 | wastage_items + documents + products | 单号,日期,品名,数量,进价,金额,原因,关联单据 |
| 巡店 | store_sales_items + documents + stores + products | 单号,店铺,日期,品名,期末库存,推算销量,单价,收入 |
| 库存 | stock_movements 聚合 | 品名,库存量 |
| 库存流水 | stock_movements + documents + products | 时间,品名,方向,数量,余额,关联单号,店铺 |
| 资金流水 | transactions + documents | 时间,类型,金额,余额,关联单号 |
| 日志 | operation_logs | 现有不变 |

### 导入列表（统一两步模式：preview 校验 + confirm 创建）

所有导入走 `csv_importer.py` 解析 → 返回校验结果 → 用户确认 → 调用 `import_confirm` 创建订单。

| 模块 | 必填列 | 可空列（默认值） | 创建逻辑 |
|------|--------|------|----------|
| 进货 | product_name, qty | unit_price→产品进价, supplier_name→"金健牛奶", date→今天, production_date→今天 | 按日期分组建单，自动 confirm |
| 零售 | product_name, qty | unit_price→客户协议价 > 等级价 > 零售价, customer_name→"散客", discount→0, date→今天 | 按 customer + date 分组建单 |
| 铺货 | product_name, qty, customer_name | unit_price→客户协议价 > 等级价 > 零售价, store_name→customer 关联店铺, discount→0, date→今天 | 按 customer + date 分组建单 |
| 退货 | product_name, qty, customer_name | unit_price→客户协议价 > 等级价 > 零售价, original_distribution_order_number→空(不关联), date→今天 | 按 customer + date 分组建单 |
| 损耗 | product_name, qty, reason | unit_price→产品进价, source_order_number→空(不关联), date→今天 | 按 date 分组建单 |
| 巡店 | product_name, actual_quantity, store_name | date→今天 | 按 store + date 分组建单 |
